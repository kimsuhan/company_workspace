# Skill Candidates

스킬화하기엔 아직 약하지만 반복되면 승격할 수 있는 관찰을 누적한다.
evolve 시 이 문서를 확인하고, 같은 패턴이 반복되면 스킬화 또는 폐기를 제안한다.

## 운영 규칙

- 후보는 최대 10개 정도만 유지한다.
- 같은 유형은 새 항목을 만들지 말고 기존 항목의 반복 횟수와 근거를 갱신한다.
- 3회 이상 반복되면 스킬화, 기존 스킬 보강, 폐기 중 하나를 제안한다.
- 60-90일 동안 반복 증거가 없으면 stale 표시 또는 삭제를 제안한다.
- 세션 요약을 길게 남기지 말고, 재사용 가능한 패턴과 승격 조건만 남긴다.

## 후보

### 시각화 UI 검증

- 상태: 관찰 중
- 처음 관찰: 2026-07-06
- 반복 횟수: 1
- 근거: DOM/canvas는 있었지만 실제 화면은 빈 차트였고, 스크린샷 확인 후에야 렌더링 실패를 발견했다.
- 스킬화 조건: 다른 세션에서도 DOM 존재 확인만으로 차트/canvas/SVG 렌더링 실패를 놓치는 일이 반복되면 별도 검증 스킬 또는 기존 verification 지침 보강을 추천한다.

### 전역 form CSS 충돌 검증

- 상태: 관찰 중
- 처음 관찰: 2026-07-06
- 반복 횟수: 2
- 근거: 전역 `.field input` 스타일이 Tiptap task checkbox에 적용되어 체크박스가 1487px로 렌더링됐다. Dashboard Layout 커스텀 number input도 전역 input 계열 스타일/선택자 우선순위 때문에 의도한 패널 UI처럼 보이지 않았다.
- 스킬화 조건: 다른 세션/프로젝트에서도 전역 form/button/label 스타일이 특정 기능 UI 내부 DOM을 깨는 문제가 반복되면 별도 검증 스킬 또는 기존 verification 지침 보강을 추천한다.
